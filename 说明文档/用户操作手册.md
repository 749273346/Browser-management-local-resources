# 浏览器本地资源管理插件 - 用户操作手册

本手册将指导您完成软件的部署、安装和日常使用。

## 第一章：部署与安装

本软件采用“绿色版”部署方式，无需复杂的安装过程，支持在无 Node.js 环境的电脑上自动配置运行。

### 1.1 部署步骤
1.  将项目文件夹完整复制到目标电脑的任意位置（建议路径不包含中文，尽管软件已做兼容处理）。
2.  双击运行根目录下的 **`start-background.bat`**。
    *   **自动环境配置**：脚本会自动检查系统是否安装 Node.js。
    *   **无网/新电脑支持**：如果未检测到 Node.js，脚本会自动尝试使用内置的离线安装包进行安装。
    *   **启动服务**：环境就绪后，脚本会自动启动后台服务。请勿关闭弹出的黑色命令行窗口（最小化即可）。

### 1.2 安装浏览器插件
后台服务启动后，请手动将插件加载到浏览器中：

#### Google Chrome 用户：
1.  在浏览器地址栏输入 `chrome://extensions` 并回车。
2.  开启右上角的 **"开发者模式"** 开关。
3.  点击左上角的 **"加载已解压的扩展程序"**。
4.  在弹出的窗口中，导航到项目目录下的 **`extension`** 文件夹，点击“选择文件夹”。

#### Microsoft Edge 用户：
1.  在浏览器地址栏输入 `edge://extensions` 并回车。
2.  开启左侧菜单底部的 **"开发人员模式"** 开关。
3.  点击 **"加载解压缩的扩展"**。
4.  在弹出的窗口中，导航到项目目录下的 **`extension`** 文件夹，点击“选择文件夹”。

## 第二章：日常使用与个性化

### 2.1 基础功能
*   **新标签页浏览**：打开浏览器新标签页即可进入资源管理界面。
*   **文件导航**：点击文件夹图标进入下级，点击顶部面包屑导航返回上级。
*   **打开文件**：点击文件图标可直接调用系统默认程序打开。

### 2.2 个性化设置 (新功能)

#### 1. 文件夹颜色标记
您可以为重要的文件夹设置醒目的颜色，以便快速识别。
*   **设置颜色**：在**一级文件夹**（根目录下的直接子文件夹）上点击**鼠标右键**，选择喜欢的颜色（如红色、蓝色、绿色等）。
*   **颜色继承**：为了保持界面整洁，**子文件夹**会自动继承其父级（一级文件夹）的颜色设置，无需逐个单独设置。
    *   *注：子文件夹无法单独修改颜色，颜色始终跟随所属的一级文件夹。*

#### 2. 壁纸管理
您可以自定义新标签页的背景壁纸。
*   **添加壁纸**：点击界面右上角的设置图标（齿轮），在“背景设置”中上传本地图片或输入图片链接。
*   **删除壁纸**：在设置面板的背景缩略图列表中，**右键点击**任意缩略图，即可将其删除。

## 第三章：常见问题 (FAQ)

**Q1: 电脑没有安装 Node.js 怎么办？**
*   **答**：无需担心。`start-background.bat` 启动脚本内置了检测和安装功能。如果检测到缺失，它会自动使用附带的离线安装包进行安装，全程无需联网。

**Q2: 打开新标签页提示“无法连接到后台服务”？**
*   **原因**：后台服务窗口被关闭或未启动。
*   **解决**：请重新运行 `start-background.bat`，并保持该窗口开启（可最小化）。如果问题依旧，请检查是否有安全软件拦截了 Node.js 进程。

**Q3: 为什么我在子文件夹上右键没有颜色选项？**
*   **答**：这是特意设计的。为了避免视觉混乱，系统强制要求子文件夹颜色必须与一级文件夹保持一致。请返回根目录，对一级文件夹进行颜色设置即可。
